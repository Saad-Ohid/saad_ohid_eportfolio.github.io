Artifact Deep Dives & Visual Proof

Category 1: Software Engineering (Inventory Mobile App)

The Challenge: The original Android application was a basic prototype that lacked user security, crashed easily during data entry, and left database connections open, which drained device resources.

The Solution: The application was completely overhauled using modern software engineering practices. A secure login screen was built using a persistent digital vault to verify users. Active session management was implemented to shred the app’s memory upon logout. To prevent crashes, safety nets were coded to handle typing mistakes gracefully. Finally, the user interface was updated with a smart bridge adapter, allowing users to delete items instantly.

Fig 1: The enhanced secure login interface preventing unauthorized access.

Category 2: Algorithms & Data Structures (Rescue Animal System)

The Challenge: The original Java program stored thousands of animal records in a basic list. Finding a specific animal required the computer to read every single record one by one, which is incredibly slow and inefficient.

The Solution: The underlying data structure was completely replaced. The slow lists were upgraded to HashMaps, functioning like a direct digital index. This algorithmic improvement dropped search times to near-zero, allowing the system to instantly retrieve data regardless of how large the database grows. Furthermore, custom logical algorithms were written to automatically cross-check an animal's training status and availability, eliminating human error when deploying rescue teams.

Fig 2: The updated Java console demonstrating role-based access control.

Category 3: Databases (Search and Rescue Dashboard)

The Challenge: The original Python and MongoDB dashboard was trapped inside a guided school server. It also had a massive security flaw: the database passwords were typed directly into the main code for anyone to see.

The Solution: The entire system was extracted and successfully deployed on an independent local server. The hardcoded passwords were removed and hidden inside a decoupled, secure configuration file. To protect the database from corrupt information, strict NoSQL schema validation was added. This ensures that any incoming data must meet specific rules before it is allowed to be saved.

Fig 3: The local deployment of the Dash/Plotly interactive database visualizer.


Mastery of Computer Science Outcomes

1. Building Collaborative Environments

Developing high-quality software requires strong collaboration and the ability to support organizational decision-making. Conducting a formal video code review demonstrated the ability to evaluate software, identify flaws, and present clear solutions to a team. Writing contextual, easy-to-read comments directly inside the source code ensures that future developers can easily understand and modify the logic. By utilizing tools like Git and GitHub, a professional environment was maintained where code changes are tracked securely, demonstrating best practices for working within modern development teams.

2. Professional Communication

Clear communication is just as important as writing functional code. Delivering a professional-quality code review video and drafting detailed technical narratives showed the ability to explain complex thought processes clearly. Technical jargon was carefully translated into easy-to-understand concepts, ensuring that both technical engineers and non-technical stakeholders (such as business managers) could understand the value of the software updates. Visual communication was also utilized by capturing system screenshots, providing clear visual proof of the application enhancements.

3. Algorithmic Principles and Problem Solving

Designing efficient computing solutions requires careful planning. Pseudocode was utilized to map out software functionality and weigh different design trade-offs before writing the actual code. For example, transitioning the rescue animal database from basic lists to HashMaps was a deliberate design choice to prioritize fast processing speed over minimal memory usage. By writing custom logic to verify animal deployment rules, complex real-world problems were successfully translated into automated, secure algorithmic solutions.

4. Innovative Techniques and Industry Standards

Bringing basic classroom projects up to industry standards required following the full Software Development Life Cycle. Iterative testing was applied continuously; for instance, the Android application was repeatedly tested with bad data to discover system crashes. Once those bugs were found, robust error handlers were built to fix them. By focusing on creating stable, crash-proof software that delivers real business value, these projects prove a strong ability to use modern computing practices to meet strict industry goals.

5. Developing a Security-First Mindset

Modern software must anticipate exploits and protect user privacy at all times. A strict "Zero Trust" security mindset was applied to every project during the design phase. Security vulnerabilities were actively hunted and eradicated—such as removing hardcoded passwords from the Python database and locking the Android application behind a secure authentication gate. All user inputs are now explicitly validated to prevent bad data from corrupting the systems. By organizing the code cleanly and keeping security credentials separated, these systems are fully prepared for future updates while keeping sensitive resources locked down.
